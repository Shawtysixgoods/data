# Граф (Graph)

## Описание структуры  
Граф — это абстрактная структура данных, которая состоит из множества вершин (узлов) и рёбер (связей), соединяющих эти вершины.  
Граф представляет отношения и связи между объектами, позволяя моделировать сложные сетевые структуры.

Основные компоненты:  
- **Вершина (узел)** — элемент графа, который может содержать данные.  
- **Ребро (дуга)** — связь между двумя вершинами, которая может иметь вес (стоимость) или метку.  
- **Вес ребра** — числовое значение, характеризующее стоимость, расстояние или интенсивность связи.

---

## Виды графов  

### По ориентированности  
- **Неориентированный граф** — рёбра не имеют направления, связь двусторонняя. Если вершина A соединена с B, то B соединена с A.  
- **Ориентированный граф (диграф)** — рёбра имеют направление, указываемое стрелкой. Связь может быть односторонней.  
- **Смешанный граф** — содержит как ориентированные, так и неориентированные рёбра.

### По наличию весов  
- **Невзвешенный граф** — рёбра не имеют весов, все связи считаются равноправными.  
- **Взвешенный граф** — каждому ребру присвоен вес (число), обозначающий расстояние, стоимость или другую характеристику.

### По структуре  
- **Связный граф** — существует путь между любыми двумя вершинами.  
- **Несвязный граф** — есть пары вершин, между которыми нет пути.  
- **Циклический граф** — содержит хотя бы один цикл (замкнутый путь).  
- **Ациклический граф** — не содержит циклов. Частный случай — ориентированный ациклический граф (DAG).  
- **Дерево** — особый вид связного ациклического графа с одной выделенной вершиной-корнем.

### Специальные типы  
- **Полный граф** — каждая вершина соединена со всеми остальными вершинами.  
- **Двудольный граф** — вершины разбиты на два множества так, что рёбра соединяют только вершины из разных множеств.  
- **Граф-решётка** — вершины расположены в виде сетки (часто используется в компьютерной графике).

## Как строится структура  

Графы можно представить несколькими способами:

### Матрица смежности  
Квадратная матрица размером N×N, где N — число вершин.  
Элемент matrix[i][j] содержит:  
- 1 (или вес), если есть ребро между вершиной i и j.  
- 0, если ребра нет.  

Преимущества: быстрый поиск ребра O(1).  
Недостатки: требует O(N²) памяти, неэффективна для разреженных графов.

### Список смежности  
Для каждой вершины хранится список вершин, с которыми она соединена.  
Обычно реализуется как словарь или массив списков.  

Преимущества: экономная по памяти O(V + E), где V — вершины, E — рёбра.  
Недостатки: поиск ребра требует O(степень вершины).

### Список рёбер  
Хранится набор рёбер в виде пар (или троек для взвешенных графов) вершин.  

Преимущества: простота реализации.  
Недостатки: менее эффективна для большинства операций.

## Применение графов  

- **Социальные сети** — вершины представляют людей, рёбра — дружеские связи.  
- **Навигация и карты** — вершины — города или перекрёстки, рёбра — дороги с расстояниями.  
- **Компьютерные сети** — вершины — компьютеры, рёбра — соединения.  
- **Рекомендательные системы** — графы интересов и взаимосвязей пользователей.  
- **Химия** — молекулы как графы, атомы как вершины, химические связи как рёбра.  
- **Алгоритмы поиска** — поиск в ширину (BFS), поиск в глубину (DFS), кратчайшие пути (Dijkstra, Bellman-Ford).

---

## Итог  

Граф — универсальная структура для моделирования различных типов связей и отношений.  
Выбор представления (матрица или список смежности) зависит от типа графа и операций, которые нужно выполнять.  
Понимание графов необходимо для решения многих задач в алгоритмике, оптимизации маршрутов и анализе данных.
